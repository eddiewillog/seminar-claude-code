# 재무팀 Claude Code 실습 가이드

> 이 문서는 **재무팀 실무자가 처음부터 끝까지 따라할 수 있도록** 만든 가이드다.
> 클로드코드(Claude Code)를 켠 상태에서, 아래 프롬프트를 **순서대로 그대로 복사·붙여넣기** 하면 된다.
>
> - 코딩 경험 없어도 된다.
> - Python 명령어를 외울 필요 없다.
> - 클로드코드가 매 단계마다 무엇을 했는지 알려준다.
> - 이상하다 싶으면 그냥 한국어로 다시 질문해도 된다.

---

## 시작 상태 — 우리는 어디서 출발하는가?

재무팀의 평소 작업 폴더는 보통 이런 모습이다. **개발자가 정리해 놓은 깔끔한 폴더가 아니다.**
그냥 여러 달치 Excel/CSV가 한 폴더에 펼쳐져 있을 뿐이다.

```
finance-practice/
├── 1월비용.xlsx
├── 2월 비용 정리.xlsx
├── 3월_비용_최종.xlsx
├── 거래처지급내역_1.csv
├── 거래처 지급 내역 2.csv
├── 임시메모.txt
└── old_backup.xlsx
```

- `data/`, `scripts/`, `reports/` 같은 폴더 ❌ 없다
- `CLAUDE.md`, `README.md` 같은 문서 ❌ 없다
- 가상환경, `requirements.txt` ❌ 없다

우리는 이 "평범한 업무 폴더"에서 클로드코드와 함께 일을 시작한다.
정리되어 가는 과정 자체가 실습이다.

---

## 0. 시작 전에 한 번만 확인하기

### 0-1. 폴더 이동

터미널에서 실습용 폴더로 들어간다.

```bash
cd 재무팀/finance-practice
```

이 안에 위에서 본 7개 파일이 들어있는지 `ls` 로 확인한다.

> 파일이 비어있거나 모양이 이상하다면, 진행자에게 "초기화"를 요청한다.
> (진행자: `cd 재무팀/_준비 && python 01_generate_messy_sample_files.py`)

### 0-2. 클로드코드 실행

같은 터미널에서 `claude` 명령으로 클로드코드를 켠다.

```bash
claude
```

이제부터는 클로드코드 입력창에 **아래 프롬프트를 하나씩** 붙여넣고 엔터를 친다.
한 단계가 끝나면 다음 단계로 넘어간다.

---

## 1단계 — 폴더 안 살펴보기

먼저 클로드코드에게 "이 폴더 뭐가 있는지 봐달라"고 부탁한다.

> **프롬프트 1:**
>
> ```
> 안녕, 나는 재무팀이야. 이 폴더 안에 어떤 파일들이 있는지 살펴보고,
> 그중에서 비용 분석에 쓸 수 있는 Excel/CSV 파일과,
> 분석에서 빼야 할 파일(백업/임시/메모 등)을 구분해서 알려줘.
> 아직 아무것도 수정하지는 말고, 보기만 한 다음 결과를 표로 보여줘.
> ```

**확인 포인트**

- 클로드코드가 `1월비용.xlsx`, `2월 비용 정리.xlsx`, `3월_비용_최종.xlsx`,
  `거래처지급내역_1.csv`, `거래처 지급 내역 2.csv` 5개를 "분석 대상"으로 분류했는지
- `old_backup.xlsx`, `임시메모.txt`는 "제외 대상"으로 분류했는지
- **원본 파일을 수정하지 않았는지**

---

## 2단계 — 안전한 작업 폴더 만들기

원본을 함부로 건드리지 않기 위해 **복사본**을 따로 두고, 작업용 폴더를 미리 만들어둔다.

> **프롬프트 2:**
>
> ```
> 좋아. 본격적으로 작업하기 전에, 원본 파일은 절대 수정하지 않도록 안전망을 만들어줘.
> 1) data/raw_snapshot/ 폴더를 만들고 원본 Excel/CSV 파일들을 그대로 복사해줘.
> 2) 앞으로 쓸 작업 폴더 data/processed/, data/output/, reports/ 도 미리 만들어줘.
> 원본 파일은 절대 옮기거나 이름을 바꾸지 마.
> ```

**확인 포인트**

- `data/raw_snapshot/` 안에 원본 파일 복사본이 들어 있는지
- `data/processed/`, `data/output/`, `reports/` 폴더가 생성됐는지
- 프로젝트 루트의 원본은 위치도 이름도 그대로인지

---

## 3단계 — 파일 점검 표 만들기

분석 대상 파일들이 각각 어떤 컬럼을 쓰고 있는지 한눈에 정리한다.

> **프롬프트 3:**
>
> ```
> 분석 대상 Excel/CSV 파일들을 하나씩 열어서 각각의 컬럼명과 행 수를 확인해줘.
> 이걸 표로 정리해서 data/processed/input_file_inventory.csv 로 저장해줘.
> 표에는 다음 컬럼이 있어야 해:
> file_name, file_type, row_count, columns, status(include 또는 exclude), note(판단 사유).
> 백업/임시 파일은 exclude 로 표시하고, 이유도 함께 적어줘.
> Excel 에서 한글이 깨지지 않게 저장해줘.
> ```

**확인 포인트**

- `data/processed/input_file_inventory.csv` 가 생성됐는지
- include 5건 / exclude 2건 으로 나뉘었는지
- Excel 로 그 파일을 열었을 때 한글이 깨지지 않는지

---

## 4단계 — 파일들 하나로 합치기 (표준화)

각 파일이 컬럼명도, 날짜 형식도, 금액 표기도 다 다르다. 클로드코드에게 통일해달라고 한다.

> **프롬프트 4:**
>
> ```
> 이제 분석 대상 파일들을 하나의 표준 형식으로 합쳐줘.
> 최종 통합 데이터의 표준 컬럼은 다음 9개로 통일해:
> transaction_date, department, vendor, category, amount, payment_method, invoice_no, memo, source_file
>
> 원본 컬럼명이 제각각이니까 (예: 거래일/date/거래 날짜, 금액/amount_krw/결제금액,
> 거래처/vendor_name/업체명 등) 알아서 잘 매핑해줘.
>
> 추가로 정리해줘야 할 것:
> - 날짜는 datetime 으로 변환
> - 금액은 쉼표/'원'/공백 같은 문자 제거하고 숫자로 변환
> - 어느 파일에서 왔는지 source_file 컬럼에 기록
> - 변환에 실패한 행은 date_parse_failed / amount_parse_failed 컬럼에 True 로 표시
> - 같은 invoice_no 가 두 번 이상 등장하면 is_duplicate_invoice 컬럼에 True 로 표시
>
> 결과는 data/processed/merged_expenses.csv 로 저장해줘. 원본은 절대 건드리지 말고.
> 끝나면 총 몇 행이 통합됐고, 변환 실패/중복은 각각 몇 건인지 요약해줘.
> ```

**확인 포인트**

- `data/processed/merged_expenses.csv` 가 생성됐는지
- 통합 행 수가 대략 90행 안팎인지 (샘플 데이터 기준)
- `date_parse_failed`, `amount_parse_failed`, `is_duplicate_invoice` 컬럼이 잘 붙어있는지
- 중복 invoice_no 가 몇 건 잡혔는지 클로드코드가 알려줬는지

---

## 5단계 — 요약 테이블 만들기 (재무팀이 자주 보는 것들)

이제 보고용 숫자를 뽑는다. 여러 개의 요약 테이블을 한 번에 부탁한다.

> **프롬프트 5:**
>
> ```
> 통합 데이터(data/processed/merged_expenses.csv)를 기반으로,
> 재무팀이 자주 보는 요약 테이블들을 만들어줘.
> 각각 CSV 로 data/output/ 폴더에 저장해.
>
> 1) monthly_expense_summary.csv  — 월(YYYY-MM)별 거래 건수와 총 금액
> 2) department_expense_summary.csv — 부서별 거래 건수와 총 금액
> 3) category_expense_summary.csv — 비용 항목별 거래 건수와 총 금액
> 4) vendor_top10_summary.csv — 거래처별 총 지출 Top 10
> 5) duplicate_invoice_list.csv — 중복 가능성이 있는 invoice_no 목록
>     (invoice_no, transaction_date, department, vendor, amount, source_file 포함)
> 6) data_quality_summary.csv — 데이터 품질 이슈 종합
>     (총 거래 건수, 날짜 변환 실패, 금액 변환 실패, 중복 invoice 건수,
>      거래처/부서/인보이스 누락 건수)
>
> 한글이 깨지지 않게 UTF-8 BOM(utf-8-sig) 으로 저장해줘.
> 정렬은 금액 큰 순서로 해줘.
> ```

**확인 포인트**

- `data/output/` 안에 CSV 6개가 다 있는지
- Excel 로 각각 열어봤을 때 한글이 깨지지 않는지
- Top 10 거래처가 보기 좋게 정렬돼 있는지

---

## 6단계 — 재무팀 보고용 Markdown 리포트 만들기

이제 위의 요약 테이블을 사람이 읽기 좋은 보고서 한 장으로 합친다.

> **프롬프트 6:**
>
> ```
> 위의 요약 테이블들과 input_file_inventory.csv 를 모아서
> 재무팀 보고용 Markdown 리포트를 만들어줘.
> 파일 위치는 reports/finance_expense_report.md
>
> 다음 섹션을 포함해줘:
> 1) 제목 + 작성일
> 2) 분석 개요 (대상 파일 수, 제외 파일 수, 통합 거래 건수, 총액)
> 3) 분석 대상 파일 목록
> 4) 제외된 파일 목록 (제외 사유 포함)
> 5) 전체 비용 합계
> 6) 월별 비용 요약 표
> 7) 부서별 비용 요약 표
> 8) 비용 항목별 요약 표
> 9) 거래처 Top 10
> 10) 데이터 품질 점검 결과
> 11) 중복 invoice_no 목록 (단정하지 말고 "중복 가능성 있음" 으로 표기)
> 12) 재무팀 확인 필요 사항 (변환 실패/중복 등 이슈를 정리)
> 13) 다음 액션
>
> 보고 톤 원칙:
> - 과장된 표현 쓰지 마.
> - 데이터로 확인되는 사실만 적어.
> - 모르거나 확실하지 않은 값은 "확인 필요" 로 표시해.
> - 중복 인보이스는 단정하지 말고 "중복 가능성 있음" 으로 표현해.
> ```

**확인 포인트**

- `reports/finance_expense_report.md` 가 생성됐는지
- 표가 깨지지 않고 잘 렌더링되는지 (Markdown 미리보기로 확인)
- 과장된 표현 없이 담백한 톤인지

## 7단계 — 작업 규칙을 문서로 남기기 (CLAUDE.md)

지금까지 클로드코드가 자기 판단으로 잘 따라줬다.
**다음 달에도 같은 규칙으로 일하게 하려면**, 그 규칙을 문서로 남겨두면 된다.
클로드코드는 `CLAUDE.md` 라는 파일이 폴더에 있으면 자동으로 읽고 따라한다.

> **프롬프트 7:**
>
> ```
> 지금까지 우리가 지킨 작업 규칙을 정리해서 CLAUDE.md 라는 파일로 만들어줘.
> 다음 달에 다시 같은 폴더에서 너(클로드코드)를 켰을 때,
> 이 파일만 읽고도 같은 방식으로 일할 수 있어야 해.
>
> 꼭 포함해야 할 규칙:
> - 이 폴더는 재무팀 비용 데이터를 정리하는 곳이다.
> - 원본 Excel/CSV 파일은 절대 수정하지 않는다.
> - 백업 복사본은 data/raw_snapshot/ 에 둔다.
> - 통합 결과는 data/processed/, 요약은 data/output/, 보고서는 reports/ 에 둔다.
> - 표준 컬럼명은 transaction_date, department, vendor, category, amount,
>   payment_method, invoice_no, memo, source_file 다.
> - 파일명에 backup/old/임시/temp 가 들어가면 분석에서 제외한다.
> - 합성 데이터/실제 데이터 처리 시 원칙은 동일하다.
> - 보고서 톤은 과장 없이 담백하게, 모르는 값은 "확인 필요".
> - 한글이 깨지지 않게 CSV는 utf-8-sig 로 저장한다.
>
> 가능하면 영어와 한국어를 섞어서 작성해줘 (코드 식별자는 영어, 사람용 설명은 한국어).
> ```

**확인 포인트**

- 폴더 루트에 `CLAUDE.md` 파일이 생겼는지
- 위 규칙들이 모두 들어있는지
- 다음 번에 클로드코드를 켜고 "지난번처럼 처리해줘" 라고만 해도 동작할지

## 8단계 — 후임자를 위한 README.md 만들기

이번엔 사람이 읽을 안내서다. 다른 동료가 이 폴더를 받았을 때 "어디서부터 봐야 하는지" 알 수 있어야 한다.

> **프롬프트 8:**
>
> ```
> 이 폴더의 README.md 를 만들어줘.
> 코딩 경험이 없는 다른 재무팀 동료가 이 폴더를 처음 받았을 때,
> 이 문서만 보고 따라할 수 있게 친절하게 적어줘.
>
> 포함할 내용:
> 1) 이 폴더가 뭐하는 곳인지 (재무 비용 데이터 정리)
> 2) 폴더 구조 (data/, reports/ 가 뭔지, 원본 파일은 어디에 두는지)
> 3) 매달 새 비용 파일이 들어오면 어떻게 해야 하는지
>    (파일을 루트에 드롭 → 클로드코드 켜서 "지난번처럼 처리해줘" 라고 말하기)
> 4) 결과물 위치 (보고서는 reports/finance_expense_report.md)
> 5) 원본 파일을 절대 수정하지 않는 이유
> 6) Google Sheets 데이터를 쓰려면? (다운로드 후 이 폴더에 넣기)
> 7) 문제가 생겼을 때 클로드코드에게 어떻게 물어볼지 (예시 프롬프트 2-3개)
>
> 너무 어려운 용어는 쓰지 말고, 사무직 동료가 읽을 수 있는 톤으로 써줘.
> ```

**확인 포인트**

- `README.md` 가 생성됐는지
- 코딩 모르는 사람이 읽어도 이해할 수 있는 톤인지
- "매달 새 파일 들어오면 이렇게 하세요" 가 명확한지

## 9단계 — (선택) 추가 실습 / 확장 과제

여기까지 따라왔다면 기본 파이프라인은 완성이다. 더 해보고 싶다면 아래 프롬프트들을 시도해본다.

### 9-1. 새로운 월 데이터 추가하기

> ```
> 4월 비용 데이터가 새로 들어왔다고 가정하고, "4월비용.xlsx" 라는 합성 데이터 파일을 만들어서
> 이 폴더에 넣어줘. 그리고 CLAUDE.md 규칙대로 다시 분석해서 리포트를 업데이트해줘.
> ```

### 9-2. 결제수단별 요약 추가

> ```
> 요약 테이블에 "결제수단별 비용 합계" 도 추가해줘.
> 파일 위치는 data/output/payment_method_summary.csv,
> 리포트에도 "결제수단별 요약" 섹션을 추가해줘.
> CLAUDE.md 에도 이 새 요약을 표준 산출물에 포함시켜줘.
> ```

### 9-3. 더 똑똑한 중복 감지

> ```
> 지금은 invoice_no 가 같은 경우만 중복으로 보고 있는데,

### Q2. 폴더가 꼬여서 처음부터 다시 하고 싶다.

진행자에게 폴더 초기화를 요청하거나, 직접 아래를 실행한다.

```bash
# 한 단계 위로 가서 _준비 스크립트 실행
cd ../_준비
python 01_generate_messy_sample_files.py
cd ../finance-practice
```

이렇게 하면 원본 Excel/CSV 파일들이 다시 처음 상태로 돌아온다.
다만 직접 만든 `data/`, `reports/`, `CLAUDE.md`, `README.md` 는 그대로 남는다.
완전 초기화하고 싶으면 클로드코드에게 부탁한다.

> ```
> data/, reports/ 폴더와 CLAUDE.md, README.md 를 다 지워서 처음 상태로 돌려줘.
> 단, finance-practice/ 안의 Excel/CSV/txt 원본은 절대 지우지 마.
> ```

### Q5. 회사 실제 데이터를 넣어도 되는지?

이 실습 프로젝트는 **합성 데이터 전용**이다. 실제 데이터로 같은 흐름을 돌리고 싶다면
별도 폴더(예: `재무팀/finance-real/`)를 만들고, 거기서 보안 정책에 맞춰 진행한다.
이 실습에서 만든 `CLAUDE.md` 를 그 폴더에 복사해두면 같은 규칙으로 동작한다.

## 마지막 한 가지

이 실습의 목표는 **클로드코드와 대화하면서 일하는 감각**을 익히는 것이다.

- 잘 모르겠으면 그냥 한국어로 물어본다.
- 결과가 이상하면 "왜 이렇게 됐어?" 라고 물어본다.
- 마음에 안 들면 "다시 해줘. 이번엔 ○○ 방식으로." 라고 말한다.
- 무서워하지 않아도 된다. 원본 파일은 백업해뒀고, 폴더는 언제든 초기화할 수 있다.

준비가 됐다면 **0단계부터** 시작하면 된다.

# Seminar — Claude Code 사내 교육 패키지

이 프로젝트는 **사내 AI Native 업무환경**을 만들기 위한 사내 교육 자료를 만드는 프로젝트다.
개발자가 아닌 실무자(인사·운영·재무 등)가 Claude Code를 자연어로 다루며,
**평소 자기 업무를 자동화·구조화하는 감각**을 갖추는 것을 목표로 한다.

---

## 우리가 만드는 것

- 부서별로 **현실적인 시작 상태**에서 출발하는 실습 패키지
- 한 명의 트레이니가 가이드만 보고 **혼자 따라할 수 있는** 단계별 프롬프트 문서
- 진행자가 워크숍을 운영할 수 있도록 한 **셋업 스크립트 + 답안 레퍼런스**
- 각 부서의 실제 반복 업무(회의록 정리, 재고 리포트, 비용 마감 등)를 그대로 옮긴 시나리오

코드 학습이 목적이 아니다. **"클로드코드와 대화하면서 일이 끝나는 경험"** 을 만드는 게 목적이다.

---

## 교육 철학

1. **비코딩 인력이 주인공이다.**
   터미널이 처음인 사람도 따라올 수 있도록, 모든 가이드는 "이 프롬프트를 그대로 붙여넣으세요" 수준에서 시작한다.

2. **정리되지 않은 폴더에서 시작한다.**
   실무자가 평소에 일하는 모습 그대로 — 이름이 제각각인 Excel 파일, 임시 메모, 오래된 백업이 섞인 상태에서 출발한다. 정리되어 가는 과정 자체가 학습이다.

3. **자기 작업 규칙을 직접 문서화한다.**
   각 실습 후반부에서 트레이니가 직접 `CLAUDE.md` 와 `README.md` 를 만든다.
   "한 번 하고 끝"이 아니라 **다음 달에도 같은 규칙으로 동작하는 자산**을 남기는 게 핵심이다.

4. **원본은 절대 건드리지 않는다.**
   감사·증빙·인계 가능성을 항상 전제로 한다. 모든 실습은 원본 보존 + 백업 복사본 + 정리본 분리 흐름을 따른다.

5. **합성 데이터만 사용한다.**
   실존 기업/거래처/개인 정보는 어떤 실습 자료에도 들어가지 않는다.

## 폴더 구성

```
seminar-claude-code/
├── README.md                   ← (이 문서) 프로젝트 전체 안내
├── 인사팀/
│   └── 회의록_정리/
│       └── 실습/
│           └── 가이드.md       ← 인사팀 회의록 자동 정리 실습
├── 운영팀/
│   ├── 회의록_정리/
│   │   └── 실습/가이드.md      ← 운영팀 회의록 정리 실습
│   └── 재고_리포트_자동화/
│       └── 실습/가이드.md      ← 주간 재고 리포트 자동화 실습
└── 재무팀/
    ├── README.md               ← 재무 패키지 전체 안내 (진행자/트레이니 공용)
    ├── 실습_가이드.md           ← 9단계 프롬프트 따라하기 가이드
    ├── finance-practice/       ← 트레이니 작업 폴더 (Excel/CSV 원본만)
    ├── _준비/                  ← 진행자용 셋업 스크립트
    └── _레퍼런스/              ← 진행자용 답안지
```

각 부서 폴더는 **독립적으로 실행 가능**하다. 다른 폴더는 신경 쓰지 않아도 된다.

---

## 현재 제공하는 실습 패키지

| 부서 | 실습 | 시작 위치 | 한 줄 설명 |
| --- | --- | --- | --- |
| 인사팀 | 회의록 정리 | `인사팀/회의록_정리/실습/가이드.md` | 이름이 제각각인 회의록 파일들을 분류·요약해 깔끔한 업무 문서로 정리 |
| 운영팀 | 회의록 정리 | `운영팀/회의록_정리/실습/가이드.md` | 운영팀 회의록 톤에 맞춘 분류·요약 흐름 |
| 운영팀 | 재고 리포트 자동화 | `운영팀/재고_리포트_자동화/실습/가이드.md` | 주간 재고 데이터를 받아 자동으로 리포트 생성 |
| 재무팀 | 비용 마감 자동화 | `재무팀/실습_가이드.md` | 흩어진 비용 Excel/CSV를 통합·요약·리포트화 + CLAUDE.md/README.md 직접 작성 |

새 실습이 추가될 때마다 이 표가 늘어난다.

## 트레이니 시작 방법 (공통)

1. 본인 부서 폴더 안에서 가이드 문서를 연다 (`가이드.md` 또는 `실습_가이드.md`).
2. 같은 폴더(또는 가이드가 지정한 작업 폴더)에서 클로드코드를 켠다.

   ```bash
   claude
   ```

3. 가이드의 **1단계부터** 프롬프트를 그대로 복사·붙여넣는다.
4. 한 단계가 끝나면 다음 단계로. 막히면 한국어로 클로드코드에게 물어본다.

> 가이드에 적힌 프롬프트는 정답이 아니라 **출발점**이다.
> 본인 업무 맥락에 맞게 단어를 바꾸거나 추가 요청을 붙여도 된다.

---

## 진행자 시작 방법

1. 워크숍 사전에 해당 부서 폴더의 README/가이드를 끝까지 읽고 한번 따라해본다.
2. 셋업 스크립트가 있는 실습은 시작 전에 한 번 실행한다.
   예: 재무팀의 경우
   ```bash
   cd 재무팀/_준비
   python 01_generate_messy_sample_files.py
   ```
3. 트레이니에게 `_레퍼런스/` 또는 답안 폴더는 **미리 보여주지 않는다**.
4. 트레이니가 막히면 "한 번에 다 하지 말고 단계별로 알려줘" 같은 보조 프롬프트를 제안한다.

## 실습 공통 환경

대부분의 실습은 아래 도구만 있으면 동작한다.

- Python 3.11+
- `pandas`, `openpyxl` (Excel/CSV 처리)
- Claude Code CLI (`claude`)

부서별 실습에 추가 의존성이 필요한 경우, 해당 폴더의 README에 명시된다.

---

## 사내 AI Native 업무환경이란

> 코드 한 줄 모르는 실무자가, 자기 업무 폴더에서 클로드코드를 켜고
> "지난번처럼 처리해줘" 라고만 말해도, 매달 똑같은 품질의 결과물이 자동으로 나오는 상태.

이를 위해 각 실습은 단순히 "한 번 자동화"가 아니라 다음 세 가지를 모두 남기도록 설계됐다.

- **데이터 파이프라인**: 원본 → 백업 → 정리 → 요약 → 보고서
- **운영 규칙 (`CLAUDE.md`)**: 다음 번 클로드코드 실행 시 자동 적용되는 작업 규약
- **인수인계 문서 (`README.md`)**: 후임자가 따라할 수 있는 안내서

이 세 가지가 한 폴더에 자리 잡으면, 그 폴더는 **사람이 바뀌어도 같은 방식으로 일하는 자산**이 된다.
그게 우리가 만드는 AI Native 업무환경의 최소 단위다.

## 확장 방향

향후 추가될 수 있는 실습 후보:

- **법무팀** — 계약서 메타데이터 자동 추출 / 만료일 추적
- **인사팀** — 채용 지원자 이력서 일괄 분류 / 면접 노트 정리
- **CS팀** — 고객 문의 로그 카테고리화 / 주간 인사이트 리포트
- **마케팅팀** — 캠페인 성과 데이터 통합 / 채널별 비교 리포트
- **임원 보고** — 부서별 산출물 통합 대시보드

새 실습을 추가할 때는 본 프로젝트의 교육 철학(현실적 시작 상태, 트레이니 주도, 원본 보존, 합성 데이터, CLAUDE.md/README.md 직접 작성) 다섯 가지를 그대로 따른다.

---

## 기여 가이드 (실습 자료를 추가/수정할 때)

1. 새 부서 폴더는 한국어 폴더명(예: `법무팀/`)으로 만든다.
2. 그 안에 트레이니가 따라할 `가이드.md` 또는 `실습_가이드.md` 를 둔다.
3. 진행자가 필요한 셋업 스크립트는 `_준비/` 에, 답안은 `_레퍼런스/` 에 둔다.
4. 새 실습이 추가되면 이 README의 **"현재 제공하는 실습 패키지"** 표에 한 줄 추가한다.
5. 모든 데이터는 합성으로 만든다. 실존 정보 사용 금지.

---

## 라이선스 및 주의

- 본 저장소의 모든 샘플 데이터는 **합성 데이터**다.
- 실제 사내 데이터로 같은 흐름을 운영할 때는 별도 폴더에서 보안 정책에 맞춰 진행한다.
- 각 실습에서 만들어진 `CLAUDE.md` 를 실제 업무 폴더에 복사해두면 같은 규칙으로 동작한다.
